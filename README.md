# Анализ результатов A/B теста дейтингового приложения

## Описание проекта
Цель – оценить успешность A/B-теста, проведённого в крупном дейтинговом приложении. В тесте для новых пользователей из нескольких стран изменялась стоимость премиум-подписки при покупке через две новые платёжные системы. Пробный период оставался прежним.

## Данные

Содержание данных: информация о пользователях и их транзакциях по группам: тестовая, контрольная группа 1 и контрольная группа 2.

***Особенности***: пропуски, дубликаты, некорректные типы данных, некорректная выгрузка логов с датами.


## Этапы анализа
1. Предобработка и очистка данных.
2. A/A тест в контрольных группах.
3. Выдвижение гипотезы для A/B-тестирования и выбор метрик для оценки результатов.
4. Расчёт метрик в группах.
5. Оценка результатов A/B-тестирования. Анализ статистической значимости различий с использованием t-критерия, z-теста, bootstrap, хи-квадрат, теста Левена, теста пропорций.
6. Формулирование выводов и предложений.

## Выводы по проекту
1. В целом эксперимент можно считать успешным, так как целевая метрика и прокси-метрика (ARPPU) показали статистически значимый рост.
2. Конверсия платных подписчиков в премиум-подписчиков хоть и упала на 16%, но это не является статистически значимым результатом. Следует отметить, что при этом конверсия среди всех пользователей осталась на прежнем уровне (статистически значимый результат).
3. Те, кто все-таки решился купить подорожавшую премиум-подписку, — это самая платежеспособная и лояльная часть аудитории. Они купили товар по более высокой цене, что и привело к значительному увеличению среднего чека. Мы отфильтровали «сомневающихся» и оставили только «основу».
4. Широкие доверительные интервалы финансовых показателей не позволяют точно оценить масштаб эффекта. Мы можем получить незначительный положительный эффект по выручке от премиум-подписок (нижняя граница доверительного интервала), но в долгосрочной перспективе это может привести к оттоку пользователей и снижению пожизненной ценности клиента.
5. Новые платежные системы могли привлечь специфическую, более платежеспособную, но узкую аудиторию.

## Предложения по дальнейшим действиям:
1. Уточнить сегментацию: проанализировать эффект по сегментам пользователей (возраст, страна, пол, тип платежной системы) для выявления наиболее чувствительных групп.
2. Проверить совместимость с другими рекламными акциями: изучить, как совместная работа новых платежных систем и будущих бонусов/скидок влияет на выручку и ARPPU.
3. Мониторинг рисков: продолжайте следить за конверсией, чтобы оперативно выявлять любые изменения, особенно при расширении теста.
4. A/B-тестирование цен в каждой платежной системе (в рамках теста) для оптимизации стоимости премиум-подписки.
5. Для принятия окончательного решения о масштабировании эксперимента необходимо оценить экономический эффект и окупаемость.
